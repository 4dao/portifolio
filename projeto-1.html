Sobre Recursos

Sobre

O Supermercado Gerim é a sua solução conveniente para compras online de supermercado. Faça suas compras com facilidade e comodidade, tudo a partir do conforto da sua casa.

Recursos

Recursos para o Cliente:

  • Opção de login para os usuários criarem e acessarem suas contas pessoais.
  • Página de perfil, proporcionando aos usuários a capacidade de alterar informações pessoais.
  • Adição e gerenciamento de endereços para entrega.
  • Área de contato para comunicação direta.
  • Histórico de Compras:
    • Visualização detalhada das compras anteriores.
    • Registro de produtos adquiridos em cada transação.
  • Sistema de Carrinho Eficiente:
    • Implementação de um carrinho de compras com validação automática.
    • Itens no carrinho por mais de 12 horas são removidos, e as quantidades são restituídas ao estoque.
    • Script interno que executa essa ação a cada intervalo de 12 horas.

    sistemas de segurança:

  • Login de Usuarios:
    • O código é desenvolvido seguindo boas práticas de segurança recomendadas no próprio documento PHP/MySQL.
    • Todas as senhas e tokens são armazenados de forma criptografada, garantindo uma camada adicional de segurança.
    • O código implementa verificações de token de sessão para garantir acesso seguro e autorizado, redirecionando para a página de login em caso de ausência ou invalidade do token.
    • A função `cpfexist` valida a existência de um CPF durante o registro, prevenindo registros duplicados no sistema.
    • No processo de registro (`registroUser`), as senhas são armazenadas de forma segura usando hash, proporcionando uma camada adicional de segurança.
    • O código demonstra práticas seguras, como o uso de consultas preparadas e filtragem de entrada para prevenir ataques potenciais.
    • Cada login é validado por meio da comparação do token, sendo este atualizado a cada novo login para fortalecer a segurança do sistema.
    • O processo de login é baseado em um CPF único, permitindo apenas uma conta associada a cada CPF. Isso assegura a singularidade das contas e reforça a integridade do sistema.
  • Compras:
    • O sistema de compras é concebido com uma lógica simples, porém robusta, visando evitar erros e acessos indevidos.
    • Apenas usuários autenticados têm permissão para realizar compras. Caso um usuário não esteja logado e deseje efetuar uma compra, automaticamente será redirecionado para a página de registro/login.
    • Após a conclusão da compra, o usuário tem acesso exclusivo aos seus próprios pedidos, rastreados por meio de identificadores únicos associados a cada usuário.
    • Um usuário pode realizar compras quantas vezes desejar até que um funcionário defina o pedido atual como "PEDIDO PREPARADO". Após essa etapa, as compras subsequentes serão tratadas separadamente.
    • No painel de administração (ADMIN), as compras dos usuários são organizadas em blocos, cada um contendo informações como endereço, detalhes do usuário e o pedido correspondente. Esses blocos são ordenados por data de chegada.
    • Após a confirmação do ADMIN como "PEDIDO PREPARADO", é atribuído um ID hash ao pedido, usado para identificar a compra atual, juntamente com o ID único do usuário. Isso garante que pedidos futuros do mesmo usuário sejam distintos e não se misturem.
    • Todas as operações realizadas para cada usuário incluem códigos de rastreamento de ID, garantindo a devida separação e segurança das operações relacionadas a cada usuário.
    • Para evitar que itens fiquem presos no estoque quando os usuários colocam produtos no carrinho, mas esquecem de finalizar a compra, o sistema executa um script. Esse script verifica itens no carrinho que permaneceram por mais de 12 horas e os retorna ao estoque, garantindo a disponibilidade adequada dos produtos.
  • Segurança Administrativa:
    • Apenas o ADMIN possui permissões para adicionar, editar e excluir informações do sistema, assegurando um controle restrito sobre operações sensíveis.
    • Cada operação que envolve informações delicadas é cuidadosamente verificada e autenticada, garantindo que apenas contas autorizadas possam realizar alterações ou acessar informações pessoais.
    • Dentre várias verificações implementadas, destaca-se uma específica responsável por assegurar que exista apenas um administrador. Caso uma tentativa seja feita para criar um novo administrador, o sistema o deleta imediatamente, preservando a singularidade da função administrativa, verificando tanto o nível quanto o ID correspondente.
    • O ADMIN não tem permissão para se excluir, sendo implementada uma verificação específica para evitar essa operação. Em caso de tentativa, o sistema exibe uma tela de erro, impedindo a exclusão do próprio administrador.

    • Estas representam algumas das robustas implementações de segurança incorporadas ao código, visando salvaguardar a integridade e segurança dos dados dos nossos clientes.
